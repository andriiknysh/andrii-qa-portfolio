# Basic Accessibility Checklist

This is a practical beginner checklist, not a formal WCAG audit.

| ID | Check | What To Look For | Result | Notes |
| --- | --- | --- | --- | --- |
| A11Y-001 | Text readability | Text is large enough to read on mobile without zooming | Not Run | [to be added] |
| A11Y-002 | Color contrast observation | Text and important buttons are readable against background colors | Not Run | [to be added] |
| A11Y-003 | Tap target size | Buttons and links are large enough to tap without precision difficulty | Not Run | [to be added] |
| A11Y-004 | Clear labels | Buttons and fields explain their purpose | Not Run | [to be added] |
| A11Y-005 | Error identification | Form errors identify the affected field and correction needed | Not Run | [to be added] |
| A11Y-006 | Keyboard/basic focus check | If using browser, key navigation does not get trapped in obvious places | Not Run | [to be added] |
| A11Y-007 | Screen orientation/layout | Content remains usable in the tested mobile orientation | Not Run | [to be added] |
| A11Y-008 | Help availability | Users can find help/support without needing to guess | Not Run | [to be added] |

## Notes

Record observations carefully. Do not claim formal accessibility certification from this checklist.

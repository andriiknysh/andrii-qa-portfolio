# Case Study 02: Mobile Usability Review

## Purpose

This case study is for reviewing a non-sensitive mobile app or public mobile website. It focuses on onboarding, signup/login, navigation, forms, labels, error messages, mobile layout, and basic accessibility checks.

## Evidence Status

Ready for real evidence collection. App or mobile site name, date, device, OS, screenshots, Loom link, and actual findings still need to be added.

## Planned Coverage

- Onboarding flow
- Signup/login entry points
- Navigation and back behavior
- Form labels and validation messages
- Primary button labels
- Mobile layout and scrolling
- Readability
- Tap-target size
- Basic color contrast observation
- Help/support visibility

Use public, low-risk apps or mobile websites where possible. Do not publish banking, payment, medical, insurance, government, or private-account screenshots unless sensitive information is fully redacted.

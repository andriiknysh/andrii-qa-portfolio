# Mobile Usability Review Plan

## Scope

Review a public mobile website or non-sensitive mobile app from a first-time-user perspective, focusing on onboarding/account setup, navigation, forms, error messages, and basic accessibility.

## Out Of Scope

- Private account workflows
- Banking, medical, insurance, or government workflows
- Real purchases
- Automated accessibility scanning
- Source-code review
- Performance testing

## Review Approach

1. Record app/site name, date, device, OS, browser or app version, and screen size.
2. Start as a new user.
3. Walk through onboarding, signup/login entry points, navigation, forms, and error states.
4. Capture screenshots only when they contain no private information or are fully redacted.
5. Record findings with impact, severity, recommendation, and evidence.

## Review Questions

- Can a first-time user understand what to do next?
- Are button labels specific?
- Are forms clearly labeled?
- Are error messages visible and useful?
- Is back navigation predictable?
- Is text readable on mobile?
- Are tap targets easy to select?
- Is help/support visible when needed?

## Tools

- Mobile device or mobile viewport
- Screenshots
- Loom
- Browser DevTools responsive mode when useful
- Markdown findings table

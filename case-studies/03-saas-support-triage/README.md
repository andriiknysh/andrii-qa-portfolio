# Case Study 03: SaaS Support Triage

## Purpose

This case study shows how a support issue can be handled from customer report to QA/product follow-up. It is meant to demonstrate clear communication, troubleshooting, escalation quality, and conversion of repeatable issues into actionable bug or product notes.

## Evidence Status

These are sample support workflows, not actual customer tickets.

## Samples

- [Support Ticket 1: Password reset email not received](support-ticket-1.md)
- [Support Ticket 2: File upload stuck in processing](support-ticket-2.md)
- [Support Ticket 3: Unexpected billing charge](support-ticket-3.md)
- [Escalation template](escalation-template.md)
- [QA/product follow-up template](qa-follow-up-template.md)

## What This Demonstrates

- Clarifying questions
- Troubleshooting steps
- Customer-facing response
- Internal escalation note
- QA/product follow-up when an issue is reproducible
- Severity and priority thinking

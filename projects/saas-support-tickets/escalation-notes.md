# SaaS Support Escalation Notes

## Escalation Template

**Issue summary:**  
Short description of what the user is experiencing.

**User impact:**  
Explain how the issue blocks or slows the user.

**Steps already tried:**  
List support troubleshooting steps completed before escalation.

**Reproduction details:**  
Browser, device, account status, timestamps, file type, or affected workflow.

**Requested engineering help:**  
State the exact question or log check needed.

## What This Demonstrates

- Structured escalation writing
- Practical product-support thinking
- Clear handoff from support to engineering

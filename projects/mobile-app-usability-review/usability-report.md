# Mobile App Usability Review

## Scope

Sample usability review for a mobile onboarding and account setup flow.

## Findings

| ID | Finding | Impact | Recommendation |
| --- | --- | --- | --- |
| UX-001 | Primary action label is unclear on the final onboarding screen | New users may not know what happens next | Use a direct label such as "Create account" or "Start setup" |
| UX-002 | Error messages disappear too quickly | Users may miss what needs to be corrected | Keep validation messages visible until the field is fixed |
| UX-003 | Password requirements are only shown after an error | Users must fail before learning the rule | Show password requirements before submission |

## What This Demonstrates

- Usability-focused testing
- Clear impact statements
- Practical product-support recommendations
